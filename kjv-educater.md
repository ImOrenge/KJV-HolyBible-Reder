# PRD: KJV 흠정역 성경전서 통독·공부 웹앱

## 1. 제품 개요

### 제품명

가칭: **KJV 리더노트**

### 제품 목적

KJV 흠정역 성경전서를 웹에서 통독하고 공부할 수 있도록 돕는 성경 리더 웹앱이다.
사용자는 성경을 읽고, 읽은 위치를 저장하고, 구절을 형광펜처럼 강조하며, 통독 진척도를 확인할 수 있다.
또한 TTS 기능을 통해 성경 구절을 음성으로 들을 수 있고, 자주 인용하는 구절을 따로 저장하여 목록화할 수 있다.

### 핵심 가치

* 성경 통독을 중단 없이 이어갈 수 있다.
* 읽은 장, 절, 위치가 자동 저장된다.
* 중요한 구절을 형광 표시하고 다시 찾아볼 수 있다.
* 자주 인용하는 구절을 주제별로 저장할 수 있다.
* 성경 본문을 눈으로 읽고 귀로 들을 수 있다.
* 개인 성경 공부 기록을 장기적으로 축적할 수 있다.

## 2. 대상 사용자

### 주요 사용자

1. KJV 흠정역 성경전서를 정기적으로 통독하려는 사용자
2. 성경 공부 중 중요한 구절을 표시하고 싶은 사용자

4. 눈으로 읽기 어려운 상황에서 성경을 TTS로 듣고 싶은 사용자
5. 성경 통독 진척도를 숫자와 시각 자료로 관리하고 싶은 사용자

### 사용자 니즈

* “내가 어디까지 읽었는지 자동으로 기억되면 좋겠다.”
* “중요한 구절을 형광펜처럼 표시하고 싶다.”
* “자주 쓰는 구절을 따로 모아두고 싶다.”
* “창세기부터 요한계시록까지 통독률을 보고 싶다.”
* “성경을 들으면서 읽고 싶다.”
* “주제별로 구절을 분류하고 싶다.”

## 4. 핵심 기능 범위

## 4.1 성경 리더

### 설명

사용자는 KJV 흠정역 성경전서를 책, 장, 절 단위로 읽을 수 있다.

### 기능 요구사항

* 성경 66권 목록 제공
* 구약/신약 구분
* 책 선택
* 장 선택
* 절 단위 본문 표시
* 이전 장 / 다음 장 이동
* 글자 크기 조절
* 줄 간격 조절
* 다크모드 지원
* 본문 보기 모드 제공

  * 일반 보기
  * 절 번호 강조 보기
  * 집중 읽기 모드

### 사용자 흐름

1. 사용자가 성경 읽기 화면에 진입한다.
2. 구약/신약 또는 성경 권별 목록에서 책을 선택한다.
3. 장을 선택한다.
4. 해당 장의 모든 절이 표시된다.
5. 사용자가 읽으면 마지막 위치가 자동 저장된다.

### 수용 기준

* 사용자는 모든 성경 권, 장, 절을 탐색할 수 있어야 한다.
* 새로고침 후에도 마지막 읽던 위치가 유지되어야 한다.
* 모바일 화면에서도 본문이 읽기 편해야 한다.

---

## 4.2 마지막 읽은 위치 저장

### 설명

사용자가 읽던 위치를 자동 저장하여 다음 접속 시 이어서 읽을 수 있도록 한다.

### 저장 기준

페이지 개념은 화면 크기와 글자 크기에 따라 달라질 수 있으므로, 기본 저장 단위는 다음과 같이 설계한다.

* 마지막 읽은 성경 권
* 마지막 읽은 장
* 마지막 읽은 절
* 스크롤 위치
* 마지막 읽은 시간

### 기능 요구사항

* 사용자가 장을 이동하면 현재 위치 자동 저장
* 사용자가 일정 시간 머무른 절을 기준으로 마지막 읽은 절 추정
* “이어 읽기” 버튼 제공
* 최근 읽은 장 목록 제공
* 사용자가 수동으로 “여기까지 읽음” 표시 가능

### 사용자 흐름

1. 사용자가 창세기 3장을 읽는다.
2. 사용자가 15절 근처까지 스크롤한다.
3. 앱은 마지막 위치를 자동 저장한다.
4. 다음 접속 시 “창세기 3장 15절부터 이어 읽기” 버튼을 보여준다.

### 수용 기준

* 사용자가 앱을 종료하고 다시 접속해도 마지막 읽은 위치가 복원되어야 한다.
* 다른 기기에서 로그인해도 동일한 위치가 동기화되어야 한다.
* 자동 저장은 사용자의 읽기 경험을 방해하지 않아야 한다.

## 4.3 통독 진척도 관리

### 설명

사용자가 성경 전체 통독률을 확인하고, 어느 책과 장을 읽었는지 관리할 수 있다.

### 기능 요구사항

* 전체 성경 통독률 표시
* 구약 통독률 표시
* 신약 통독률 표시
* 권별 통독률 표시
* 장별 읽음 상태 표시
* 오늘 읽은 장 수 표시
* 총 읽은 장 수 표시
* 연속 읽기 일수 표시
* “읽음 처리” 버튼 제공
* “읽음 취소” 기능 제공

### 진척도 계산 기준

* 성경 전체 장 수 기준으로 계산
* 장 단위 읽음 완료 시 진척률 반영
* 선택적으로 절 단위 진척률 확장 가능

### 예시

* 전체 통독률: 18.4%
* 구약 통독률: 12.1%
* 신약 통독률: 36.8%
* 오늘 읽은 장: 5장
* 연속 통독일: 7일

### 사용자 흐름

1. 사용자가 출애굽기 12장을 끝까지 읽는다.
2. “이 장 읽음 완료” 버튼을 누른다.
3. 출애굽기 12장이 읽음 처리된다.
4. 통독률이 자동 업데이트된다.
5. 대시보드에서 진척도를 확인한다.

### 수용 기준

* 사용자는 자신이 읽은 장과 읽지 않은 장을 명확히 구분할 수 있어야 한다.
* 통독률은 실시간으로 반영되어야 한다.
* 읽음 처리한 장은 다시 읽음 취소할 수 있어야 한다.

## 4.4 구절 형광 강조 표시

### 설명

사용자는 중요한 구절을 형광펜처럼 강조할 수 있다.

### 기능 요구사항

* 절 단위 강조 표시
* 여러 색상 제공

  * 노란색: 중요
  * 파란색: 묵상
  * 초록색: 약속
  * 빨간색: 경고
  * 보라색: 예언
* 강조 표시 해제
* 강조된 구절 목록 보기
* 강조한 날짜 저장
* 강조 색상 변경
* 강조 구절에 짧은 메모 추가 가능

### 사용자 흐름

1. 사용자가 요한복음 3장 16절을 선택한다.
2. 하단 메뉴에서 “강조”를 누른다.
3. 색상을 선택한다.
4. 해당 구절이 형광 표시된다.
5. 강조 구절 목록에서 다시 확인할 수 있다.

### 수용 기준

* 강조한 구절은 다시 접속해도 유지되어야 한다.
* 한 구절에 하나 이상의 태그 또는 메모를 연결할 수 있어야 한다.
* 강조 색상은 사용자가 변경할 수 있어야 한다.

## 4.5 자주 인용하는 구절 저장

### 설명

사용자는 자주 인용하는 구절을 별도 목록에 저장할 수 있다.

### 기능 요구사항

* 구절 저장 버튼 제공
* 저장된 구절 목록 페이지 제공
* 구절 제목 또는 별칭 설정
* 주제별 분류
* 태그 추가
* 구절 복사 기능
* 구절 공유 기능
* 목록 내 검색
* 정렬 기능

  * 최근 저장순
  * 성경 순서
  * 자주 사용순
  * 태그별

### 예시 태그

* 구원
* 믿음
* 창조
* 심판
* 지혜
* 회개
* 기도
* 위로
* 예언
* 복음
* 전도
* 종말
* 하나님의 말씀

### 사용자 흐름

1. 사용자가 로마서 10장 9절을 선택한다.
2. “인용 구절에 저장”을 누른다.
3. 태그로 “구원”, “복음”을 추가한다.
4. 나중에 “내 인용 구절” 페이지에서 확인한다.
5. 버튼 하나로 구절을 복사한다.

### 수용 기준

* 사용자는 원하는 구절을 인용 목록에 저장할 수 있어야 한다.
* 저장된 구절은 태그와 함께 관리할 수 있어야 한다.
* 구절 복사 시 성경 위치가 함께 복사되어야 한다.

### 복사 형식 예시

“태초에 하나님께서 하늘과 땅을 창조하시니라.”
창세기 1:1, 킹제임스 흠정역

## 4.6 TTS 성경 읽기

### 설명

사용자는 성경 본문을 TTS로 들을 수 있다.

### 기능 요구사항

* 현재 장 전체 읽기
* 선택한 구절만 읽기
* 선택한 구절 범위 읽기
* 재생 / 일시정지 / 정지
* 이전 절 / 다음 절 이동
* 읽는 속도 조절

  * 0.75x
  * 1.0x
  * 1.25x
  * 1.5x
* 음성 선택

  * 브라우저 기본 TTS
  * 서버 기반 고품질 TTS는 추후 확장
* 현재 읽고 있는 절 하이라이트
* 화면 꺼짐 방지 또는 백그라운드 재생은 앱 확장 단계에서 검토

### 사용자 흐름

1. 사용자가 시편 23장을 연다.
2. “음성으로 듣기” 버튼을 누른다.
3. TTS가 1절부터 읽기 시작한다.
4. 현재 읽고 있는 절이 자동으로 강조된다.
5. 사용자는 속도를 조절하거나 일시정지할 수 있다.

### 수용 기준

* 사용자는 현재 장을 음성으로 들을 수 있어야 한다.
* TTS 재생 중 현재 절이 시각적으로 표시되어야 한다.
* 모바일 브라우저에서도 기본 재생 기능이 동작해야 한다.

## 4.7 검색 기능

### 설명

사용자는 성경 본문에서 단어 또는 구절을 검색할 수 있다.

### 기능 요구사항

* 전체 성경 검색
* 구약/신약 필터
* 성경 권별 필터
* 정확한 구절 검색
* 키워드 검색
* 검색 결과에서 해당 장으로 이동
* 검색 결과 내 구절 저장/강조 가능

### 사용자 흐름

1. 사용자가 “믿음”을 검색한다.
2. 검색 결과에 관련 구절 목록이 표시된다.
3. 사용자가 원하는 구절을 클릭한다.
4. 해당 장과 절로 이동한다.

### 수용 기준

* 검색 결과는 빠르게 표시되어야 한다.
* 검색 결과에서 바로 구절 저장, 복사, 강조가 가능해야 한다.

## 12. 저작권 및 라이선스 체크

성경 본문을 앱 내부 DB에 저장하고 서비스하려면, KJV 흠정역 본문 사용 권한을 반드시 확인해야 한다.

확인 항목:

* 본문 DB 저장 가능 여부
* 웹 서비스 공개 가능 여부
* 검색 기능 제공 가능 여부
* TTS 낭독 제공 가능 여부
* 구절 복사 및 공유 가능 범위
* 상업적 서비스 가능 여부
* 출처 표기 방식

라이선스가 명확하지 않다면 MVP 단계에서는 다음 중 하나를 선택한다.

1. 사용자가 직접 본문 파일을 업로드하는 개인용 앱 구조
2. 허가받은 본문 데이터만 사용
3. 외부 링크 기반 읽기 보조 앱으로 시작
4. 본문 없는 통독 기록·구절 관리 앱으로 먼저 출시

## 13. 핵심 사용자 시나리오

## 시나리오 1: 통독 이어 읽기

1. 사용자가 로그인한다.
2. 홈에서 “이어서 읽기”를 누른다.
3. 마지막으로 읽던 성경 장이 열린다.
4. 사용자가 계속 읽는다.
5. 장을 다 읽고 “읽음 완료”를 누른다.
6. 통독률이 업데이트된다.

## 시나리오 2: 구절 강조

1. 사용자가 중요한 구절을 발견한다.
2. 해당 절을 클릭한다.
3. “강조”를 누른다.
4. 색상을 선택한다.
5. 구절이 형광 표시된다.
6. 강조 목록에서 다시 확인한다.

## 시나리오 3: 자주 인용하는 구절 저장

1. 사용자가 자주 쓰는 구절을 선택한다.
2. “인용 구절에 저장”을 누른다.
3. 제목과 태그를 입력한다.
4. 저장된 구절 목록에서 확인한다.
5. 복사 버튼으로 본문과 출처를 복사한다.

## 시나리오 4: TTS로 듣기

1. 사용자가 특정 장을 연다.
2. “듣기” 버튼을 누른다.
3. TTS가 현재 장을 읽는다.
4. 현재 읽고 있는 절이 하이라이트된다.
5. 사용자는 속도를 조절하거나 일시정지한다.

## 17. 리스크

## 17.1 성경 본문 라이선스

가장 중요한 리스크는 KJV 흠정역 본문 사용 권한이다.
앱에 본문을 직접 저장하고 제공할 경우, 배포 허가와 사용 조건을 반드시 확인해야 한다.

## 17.2 TTS 품질

브라우저 기본 TTS는 기기와 브라우저마다 음성 품질이 다르다.
MVP에서는 기본 TTS를 사용하고, 추후 고품질 TTS를 도입한다.

## 17.3 페이지 저장 문제

웹에서는 화면 크기와 글자 크기에 따라 페이지가 달라진다.
따라서 실제 저장 기준은 페이지가 아니라 성경 권, 장, 절, 스크롤 위치로 관리해야 한다.

## 17.4 데이터 동기화

여러 기기에서 읽기 기록이 충돌할 수 있다.
최신 저장 시간을 기준으로 동기화하되, 수동 읽음 처리는 별도로 보존한다.

## 18. 최종 MVP 정의

이 웹앱의 첫 버전은 “성경 통독을 이어가고, 중요한 구절을 표시하고, 자주 인용하는 구절을 저장하며, TTS로 들을 수 있는 개인 성경 공부 도구”로 정의한다.

MVP의 핵심은 화려한 기능보다 다음 네 가지다.

1. 읽던 곳을 잃어버리지 않게 한다.
2. 통독 진척도를 명확히 보여준다.
3. 중요한 구절을 형광 표시하고 다시 찾게 한다.
4. 자주 인용하는 구절을 개인 보관함처럼 관리하게 한다.

이 네 가지가 안정적으로 작동하면, 이후 통독 플랜, 암송 모드, 구절 이미지 생성, 고품질 TTS, 성경 공부 노트 기능으로 확장한다.
