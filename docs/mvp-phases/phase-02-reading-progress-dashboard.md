# Phase 2: 통독 기능 및 대시보드

## 목표

사용자가 읽던 위치를 잃어버리지 않고 이어서 읽을 수 있게 하며, 장 단위 읽음 완료와 통독률을 mock 사용자 기준으로 관리한다. 이 단계가 끝나면 앱은 단순 리더가 아니라 통독 진행을 추적하는 개인 도구가 된다.

## 구현 체크리스트

### 1. 데이터 구조

- [ ] `reading_progress` 테이블 또는 동등한 저장소를 만든다.
- [ ] `completed_chapters` 테이블 또는 동등한 저장소를 만든다.
- [ ] `user_id`, `book_id`, `chapter`, `verse`, `scroll_position`, `last_read_at`을 저장한다.
- [ ] 사용자별 같은 권/장 중복 완료 처리를 방지한다.
- [ ] `completed_chapters` 조회 성능을 위한 인덱스를 추가한다.
- [ ] 여러 기기 충돌 시 `last_read_at` 기준 병합 규칙을 정한다.

### 2. 읽기 기록 API

- [ ] `GET /api/me/progress`를 구현한다.
- [ ] `POST /api/me/progress`를 구현한다.
- [ ] `POST /api/me/completed-chapters`를 구현한다.
- [ ] `DELETE /api/me/completed-chapters/:id` 또는 권/장 기반 취소 API를 구현한다.
- [ ] 모든 API에서 인증을 요구한다.
- [ ] 모든 API에서 요청 사용자의 데이터만 조회/변경되도록 검증한다.
- [ ] 잘못된 권/장/절 입력을 검증한다.

### 3. 마지막 읽은 위치 저장

- [ ] 장 진입 시 현재 권/장 정보를 저장한다.
- [ ] 스크롤 위치를 저장한다.
- [ ] 화면에 오래 머문 절을 기준으로 마지막 읽은 절을 추정한다.
- [ ] 저장 요청은 debounce 처리해 과도한 API 호출을 막는다.
- [ ] 저장 실패 시 읽기 경험을 방해하지 않는 방식으로 처리한다.
- [ ] 새로고침 후 마지막 권/장/절/스크롤 위치를 복원한다.
- [ ] 같은 브라우저에서 새로고침 후에도 최신 위치를 불러온다.

### 4. 이어 읽기 UX

- [ ] 홈 상단에 "이어서 읽기" 카드를 만든다.
- [ ] 카드에 성경 권, 장, 절, 마지막 읽은 시간을 표시한다.
- [ ] 버튼 클릭 시 해당 장과 절 근처로 이동한다.
- [ ] 최근 읽은 장 목록을 만든다.
- [ ] 수동 "여기까지 읽음" 표시를 제공한다.

### 5. 읽음 완료 처리

- [ ] 성경 읽기 화면에 "이 장 읽음 완료" 버튼을 배치한다.
- [ ] 완료된 장은 리더와 진척도 화면에서 표시한다.
- [ ] 완료된 장은 다시 "읽음 취소"할 수 있게 한다.
- [ ] 장 완료 시 통독률을 즉시 갱신한다.
- [ ] 중복 클릭과 네트워크 지연에 대한 로딩/비활성 상태를 처리한다.

### 6. 통독률 계산

- [ ] 성경 전체 장 수 기준 통독률을 계산한다.
- [ ] 구약 장 수 기준 통독률을 계산한다.
- [ ] 신약 장 수 기준 통독률을 계산한다.
- [ ] 권별 통독률을 계산한다.
- [ ] 오늘 읽은 장 수를 계산한다.
- [ ] 총 읽은 장 수를 계산한다.
- [ ] 연속 읽기 일수를 계산한다.
- [ ] 계산 로직을 서버 또는 공유 유틸로 분리해 UI와 중복을 줄인다.

### 7. 대시보드 및 진척도 화면

- [ ] 대시보드에 이어 읽기 카드를 배치한다.
- [ ] 전체 통독률을 숫자와 시각 요소로 표시한다.
- [ ] 오늘 읽은 장 수와 총 읽은 장 수를 표시한다.
- [ ] 연속 통독일을 표시한다.
- [ ] 최근 읽은 성경을 표시한다.
- [ ] 권별 진척도 목록을 만든다.
- [ ] 장별 읽음 체크 UI를 만든다.
- [ ] 통독 진척도 화면에서 읽음 완료/취소를 직접 처리할 수 있게 한다.

### 8. 검증

- [ ] 창세기 1장을 읽음 완료하면 전체 통독률이 증가한다.
- [ ] 읽음 취소를 하면 통독률이 감소한다.
- [ ] 앱을 새로고침해도 마지막 읽은 위치가 복원된다.
- [ ] 같은 브라우저에서 mock 사용자로 다시 진입하면 최신 위치가 보인다.
- [ ] 완료 장 중복 저장이 발생하지 않는다.
- [ ] 모바일에서 이어 읽기와 읽음 완료 버튼이 사용 가능하다.

## 수용 기준

- 사용자가 앱을 종료하고 다시 접속해도 마지막 읽은 위치가 복원된다.
- 사용자는 장 단위 읽음 완료와 읽음 취소를 할 수 있다.
- 전체, 구약, 신약, 권별 통독률이 실시간으로 반영된다.
- 대시보드에서 사용자는 다음에 무엇을 읽어야 하는지 바로 알 수 있다.
- v0에서는 mock 사용자 기준으로 읽기 기록이 유지되고, 실제 출시 전에는 사용자별 읽기 기록 분리를 적용한다.
