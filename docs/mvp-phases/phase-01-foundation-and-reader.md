# Phase 1: 기초 구조 및 성경 리더

## 목표

웹앱의 기본 구조를 세우고, mock 로그인 사용자가 성경 권, 장, 절 단위로 본문을 탐색할 수 있는 리더를 구현한다. 이 단계의 결과물은 이후 통독 기록, 강조, 인용 구절, TTS 기능이 붙을 수 있는 안정적인 기반이어야 한다.

## PRD 근거

- 3. 제품 목표: 성경 본문 읽기, 사용자 계정 기반 데이터 동기화
- 4.1 성경 리더
- 4.9 사용자 계정 및 동기화
- 5.2 성경 읽기 화면
- 6.1-6.3 Users, Bible Books, Bible Verses
- 7.1 성경 본문 API
- 8. MVP 필수: 회원가입/로그인, 성경 권/장/절 보기, 글자 크기/다크모드 설정
- 현재 구현 전제: OAuth는 mock session으로 두고, 성경전서 전체 본문 삽입은 DB 구현 단계 이후로 미룬다.

## 포함 범위

- 프로젝트 초기 세팅
- mock 인증 및 사용자 프로필 기반
- 성경 권/장 메타데이터와 일부 fixture 본문
- 성경 리더 UI
- 기본 설정 UI
- 반응형 레이아웃
- 본문 라이선스 처리 방침 결정

## 제외 범위

- 마지막 읽은 위치 저장
- 읽음 완료 및 통독률 계산
- 구절 강조와 인용 구절 저장
- TTS 재생
- 검색
- 배포 운영 준비

## 구현 체크리스트

### 1. 프로젝트 기반

- [ ] Next.js, React, TypeScript 기반 프로젝트를 생성한다.
- [ ] Tailwind CSS 또는 선택한 스타일 시스템을 설정한다.
- [ ] ESLint, Prettier, TypeScript strict 옵션을 설정한다.
- [ ] 환경 변수 구조를 정리한다.
- [ ] 로컬 개발 실행 명령과 빌드 명령을 문서화한다.
- [ ] 기본 라우팅 구조를 만든다.

### 2. Mock 인증 기반

- [ ] OAuth mock 로그인 화면을 만든다.
- [ ] `demo-user` mock session을 만든다.
- [ ] 로그아웃 기능을 만든다.
- [ ] mock 사용자 세션을 앱 전역에서 확인할 수 있게 한다.
- [ ] 비로그인 상태의 사용자가 개인 데이터 화면에 접근할 때 mock 로그인으로 유도한다.
- [ ] 실제 OAuth 전환을 위해 인증 인터페이스를 분리한다.
- [ ] 실제 provider callback, token refresh, 계정 연결은 후속 DB/Auth 전환 단계로 남긴다.

### 3. 성경 데이터 모델

- [ ] `bible_books` fixture 또는 동등한 데이터 구조를 만든다.
- [ ] 일부 장/절만 포함한 `bible_verses` fixture 또는 동등한 데이터 구조를 만든다.
- [ ] 구약/신약 구분 필드를 정의한다.
- [ ] 성경 권 순서, 권 이름, 영문 이름, 장 수를 저장한다.
- [ ] 장과 절을 안정적으로 조회할 수 있는 repository 인터페이스를 설계한다.
- [ ] 본문 데이터 출처와 라이선스 상태를 문서화한다.
- [ ] 성경전서 전체 본문 삽입은 DB 구현 단계 이후 작업으로 명시한다.
- [ ] fixture에 없는 장은 "본문 준비 중" 상태로 처리한다.

### 4. 성경 본문 API

- [ ] `GET /api/bible/books`를 구현한다.
- [ ] `GET /api/bible/books/:bookId/chapters`를 구현한다.
- [ ] `GET /api/bible/books/:bookId/chapters/:chapter`를 구현한다.
- [ ] 잘못된 권, 장 요청에 대한 404 또는 검증 오류를 처리한다.
- [ ] 성경 장 로딩이 1초 이내를 목표로 동작하는지 확인한다.
- [ ] API 응답 형태를 프론트엔드 컴포넌트와 맞춘다.

### 5. 성경 리더 UI

- [ ] 홈 또는 대시보드에서 성경 읽기 화면으로 이동할 수 있게 한다.
- [ ] 구약/신약 구분 목록을 제공한다.
- [ ] 66권 성경 권 목록을 제공한다.
- [ ] 선택한 권의 장 목록을 제공한다.
- [ ] 장 선택 시 절 단위 본문을 표시한다.
- [ ] 이전 장/다음 장 이동을 구현한다.
- [ ] 절 번호와 본문 텍스트를 명확히 구분한다.
- [ ] 모바일 화면에서 본문 줄 길이와 터치 영역을 조정한다.

### 6. 기본 보기 설정

- [ ] 글자 크기 조절 UI를 만든다.
- [ ] 줄 간격 조절 UI를 만든다.
- [ ] 다크모드 토글을 만든다.
- [ ] 일반 보기 모드를 구현한다.
- [ ] 절 번호 강조 보기 모드를 구현한다.
- [ ] 집중 읽기 모드의 최소 UI를 구현한다.
- [ ] 설정값을 우선 로컬에 저장하고, 로그인 후 계정 설정과 연동할 구조를 마련한다.

### 7. 검증

- [ ] mock 로그인 후 성경 읽기 화면에 접근할 수 있다.
- [ ] 모든 성경 권 목록이 올바른 순서로 표시된다.
- [ ] 선택한 권의 장 수가 PRD 데이터와 맞는다.
- [ ] 장 이동 시 본문이 절 단위로 표시된다.
- [ ] 이전 장/다음 장 이동이 권 경계를 처리한다.
- [ ] 글자 크기, 줄 간격, 다크모드가 리더 화면에 반영된다.
- [ ] 데스크톱, 태블릿, 모바일 폭에서 본문이 깨지지 않는다.

## 수용 기준

- 사용자는 mock 로그인 후 성경 권, 장, 절을 탐색할 수 있다.
- 사용자는 성경 66권과 각 장을 선택할 수 있다.
- 사용자는 모바일 화면에서도 본문을 읽기 편하다.
- 글자 크기와 다크모드 설정이 리더 화면에 반영된다.
- 전체 본문 DB 삽입이 후속 단계임이 문서화되어 있다.
