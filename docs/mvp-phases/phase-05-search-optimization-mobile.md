# Phase 5: 검색, 최적화, 모바일 사용성

## 목표

성경 본문과 저장 데이터를 빠르게 찾을 수 있게 하고, MVP 출시 전 모바일 사용성과 성능을 보강한다. PRD의 개발 단계에는 포함되어 있으나, "MVP에 반드시 포함할 기능" 목록에는 검색이 없으므로 일정에 따라 P1 후속 범위로 분리할 수 있다.

## 구현 체크리스트

### 1. 검색 정책

- [ ] MVP 출시 전 검색을 포함할지 P1 후속으로 뺄지 결정한다.
- [ ] 검색 대상 범위를 성경 본문, 강조 구절, 인용 구절로 나눈다.
- [ ] 정확한 구절 검색과 키워드 검색의 입력 규칙을 정의한다.
- [ ] 검색 결과 정렬 기준을 정의한다.
- [ ] 검색어 최소 길이와 빈 검색 처리 정책을 정한다.

### 2. 성경 본문 검색 API

- [ ] `GET /api/bible/search?q=keyword`를 구현한다.
- [ ] 구약/신약 필터를 추가한다.
- [ ] 성경 권별 필터를 추가한다.
- [ ] 장/절 위치와 본문 일부를 함께 반환한다.
- [ ] 검색 결과에서 해당 장으로 이동할 수 있는 식별자를 반환한다.
- [ ] 검색 결과는 2초 이내 표시를 목표로 최적화한다.
- [ ] PostgreSQL Full Text Search 또는 적합한 인덱스 전략을 검토한다.

### 3. 개인 데이터 검색과 필터

- [ ] 강조 구절 목록에서 색상 필터를 개선한다.
- [ ] 강조 구절 목록에서 성경 권 필터를 개선한다.
- [ ] 인용 구절 목록에서 태그 필터를 제공한다.
- [ ] 인용 구절 목록 내 키워드 검색을 제공한다.
- [ ] 최근 저장순, 성경 순서, 자주 사용순, 태그별 정렬을 정리한다.
- [ ] 필터 상태를 URL 또는 화면 상태로 유지할지 결정한다.

### 4. 검색 화면 UI

- [ ] 검색 입력창을 만든다.
- [ ] 검색 범위 필터를 만든다.
- [ ] 검색 결과 목록을 만든다.
- [ ] 결과 항목에 성경 위치를 명확히 표시한다.
- [ ] 결과 항목에서 장으로 이동할 수 있게 한다.
- [ ] 결과 항목에서 구절 저장을 할 수 있게 한다.
- [ ] 결과 항목에서 구절 강조를 할 수 있게 한다.
- [ ] 결과 항목에서 구절 복사를 할 수 있게 한다.
- [ ] 검색 중 로딩, 결과 없음, 오류 상태를 처리한다.

### 5. 성능 최적화

- [ ] 성경 장 로딩이 1초 이내인지 측정한다.
- [ ] 검색 결과가 2초 이내인지 측정한다.
- [ ] 대시보드 로딩이 2초 이내인지 측정한다.
- [ ] 큰 목록 렌더링 비용을 점검한다.
- [ ] 불필요한 클라이언트 fetch와 재렌더링을 줄인다.
- [ ] 자주 쓰는 성경 메타데이터 캐싱 전략을 정한다.
- [ ] API 응답 크기를 점검한다.

### 6. 모바일 사용성

- [ ] 성경 권/장 선택 UI가 모바일에서 빠르게 조작되는지 확인한다.
- [ ] 절 선택 메뉴가 작은 화면에서 잘리지 않게 한다.
- [ ] 하단 액션 메뉴가 본문을 과도하게 가리지 않게 한다.
- [ ] 강조 색상 선택이 터치하기 쉬운 크기인지 확인한다.
- [ ] TTS 플레이어 버튼 간격을 조정한다.
- [ ] 대시보드 주요 수치가 한 화면에서 읽히는지 확인한다.
- [ ] 검색 결과 항목의 터치 영역을 충분히 확보한다.

### 7. 접근성 보강

- [ ] 키보드로 성경 권/장 선택이 가능하다.
- [ ] 주요 버튼에 접근 가능한 이름을 제공한다.
- [ ] 다크모드 대비를 점검한다.
- [ ] 강조 색상에만 의존하지 않는 상태 표시를 추가한다.
- [ ] 글자 크기 조절 후 레이아웃 깨짐을 확인한다.
- [ ] TTS 컨트롤이 스크린리더에서 이해 가능하도록 라벨을 제공한다.

### 8. 검증

- [ ] "믿음" 검색 결과가 2초 이내에 표시된다.
- [ ] 검색 결과에서 해당 장과 절로 이동할 수 있다.
- [ ] 검색 결과에서 구절 저장, 강조, 복사가 가능하다.
- [ ] 인용 구절 보관함에서 태그 필터가 동작한다.
- [ ] 모바일 폭에서 리더, 대시보드, 보관함, 검색 화면이 겹치지 않는다.
- [ ] 글자 크기를 키워도 주요 버튼 텍스트가 잘리지 않는다.

## 수용 기준

- 사용자는 성경 본문에서 단어 또는 구절을 검색할 수 있다.
- 검색 결과에서 바로 해당 장으로 이동할 수 있다.
- 검색 결과에서 저장, 강조, 복사 작업을 할 수 있다.
- 검색 결과는 목표 시간 내에 표시된다.
- 모바일 화면에서 핵심 흐름이 터치하기 쉽고 깨지지 않는다.
