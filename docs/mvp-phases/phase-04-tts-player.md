# Phase 4: 기본 TTS 성경 읽기

## 목표

브라우저 Web Speech API를 사용해 현재 장 또는 선택 구절을 음성으로 들을 수 있게 한다. MVP에서는 서버 기반 고품질 TTS를 제외하고, 사용자가 읽는 위치를 시각적으로 따라갈 수 있는 기본 낭독 경험에 집중한다.

## 포함 범위

- 현재 장 전체 읽기
- 선택 구절 읽기
- 선택 구절 범위 읽기
- 재생/일시정지/정지
- 이전 절/다음 절 이동
- 속도 조절

- TTS 설정 저장

## 제외 범위

- 서버 기반 고품질 TTS
- 음성 파일 캐싱
- 장 단위 오디오 생성
- 화면 꺼짐 방지
- 백그라운드 재생 보장
- 네이티브 모바일 오디오 기능

## 구현 체크리스트

### 1. TTS 서비스 구조

- [ ] Web Speech API 사용 가능 여부를 감지한다.
- [ ] 지원하지 않는 브라우저에 대한 안내 상태를 만든다.
- [ ] 구절 배열을 받아 순차 재생하는 TTS 서비스 모듈을 만든다.
- [ ] 음성 목록 로딩과 변경 이벤트를 처리한다.
- [ ] 컴포넌트 언마운트 시 speech synthesis를 정리한다.
- [ ] 장 이동 시 이전 TTS 재생을 안전하게 중지한다.

### 2. 재생 대상

- [ ] 현재 장 전체를 1절부터 읽을 수 있게 한다.
- [ ] 선택한 단일 구절만 읽을 수 있게 한다.
- [ ] 선택한 구절 범위를 읽을 수 있게 한다.
- [ ] 현재 절 이후부터 이어 듣기 동작을 검토한다.
- [ ] 성경 위치와 본문을 TTS queue에 안정적으로 매핑한다.

### 3. 플레이어 UI

- [ ] 성경 읽기 화면에 TTS 버튼을 배치한다.
- [ ] 재생 버튼을 만든다.
- [ ] 일시정지 버튼을 만든다.
- [ ] 정지 버튼을 만든다.
- [ ] 이전 절 버튼을 만든다.
- [ ] 다음 절 버튼을 만든다.
- [ ] 재생 중인 절 번호를 표시한다.
- [ ] 로딩, 재생 중, 일시정지, 종료 상태를 구분한다.

### 4. 속도와 음성 설정

- [ ] 0.75x 속도를 제공한다.
- [ ] 1.0x 속도를 제공한다.
- [ ] 1.25x 속도를 제공한다.
- [ ] 1.5x 속도를 제공한다.
- [ ] 브라우저가 제공하는 음성 목록을 선택할 수 있게 한다.
- [ ] 사용자의 TTS 속도 설정을 저장한다.
- [ ] 사용자의 TTS 음성 설정을 저장한다.
- [ ] 설정 화면과 리더 플레이어의 값이 동기화되게 한다.

### 5. 현재 절 하이라이트

- [ ] TTS 재생 중 현재 읽는 절을 시각적으로 표시한다.
- [ ] 현재 절이 화면 밖이면 부드럽게 스크롤할지 정책을 정한다.
- [ ] 사용자가 수동 스크롤 중일 때 자동 스크롤이 방해하지 않도록 조정한다.
- [ ] 재생 종료 시 하이라이트 상태를 정리한다.
- [ ] 강조 색상과 TTS 하이라이트가 충돌하지 않는 스타일 규칙을 만든다.

### 6. 브라우저 제약 대응

- [ ] 모바일 브라우저에서 사용자 제스처 이후 재생을 시작한다.
- [ ] 음성 목록이 늦게 로딩되는 브라우저를 처리한다.
- [ ] 일시정지/재개 동작이 브라우저별로 다를 수 있음을 검증한다.
- [ ] 긴 장 재생 중 중단되거나 끊기는 경우 재시작 전략을 마련한다.
- [ ] TTS가 실패해도 리더 자체 기능은 계속 사용할 수 있게 한다.

### 7. 검증

- [ ] 시편 23장을 장 전체로 재생할 수 있다.
- [ ] 선택한 단일 구절만 재생할 수 있다.
- [ ] 선택한 구절 범위를 순서대로 재생할 수 있다.
- [ ] 재생/일시정지/정지 버튼이 동작한다.
- [ ] 이전 절/다음 절 이동이 동작한다.
- [ ] 속도 변경이 다음 재생에 반영된다.
- [ ] 현재 읽는 절이 화면에서 표시된다.
- [ ] 모바일 브라우저에서 기본 재생 흐름이 동작한다.

## 수용 기준

- 사용자는 현재 장을 음성으로 들을 수 있다.
- 사용자는 선택한 구절 또는 구절 범위를 들을 수 있다.
- TTS 재생 중 현재 절이 시각적으로 표시된다.
- 사용자는 재생, 일시정지, 정지, 속도 조절을 사용할 수 있다.
- TTS 설정은 계정 또는 사용자 설정으로 유지된다.
- 지원하지 않는 브라우저에서도 앱이 깨지지 않는다.

## 산출물

- 리더 내 TTS 플레이어
- 선택 구절 TTS 연동
- 속도/음성 설정 UI
- 현재 절 하이라이트 스타일
- 브라우저 지원 fallback 상태
