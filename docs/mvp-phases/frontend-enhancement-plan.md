# 프론트엔드 기능 보강 계획

이 문서는 현재 v0 MVP 위에 추가할 프론트엔드 기능 계획이다. 이전 후보 기능 중 7번 `데이터 내보내기/가져오기`는 이번 범위에서 제외한다.

## 현재 전제

- 계정 OAuth는 계속 mock session으로 둔다.
- 개인 데이터는 현재 구조처럼 localStorage 기반으로 검증한다.
- 성경전서 전체 본문 seed와 운영 DB 전환은 별도 DB 구현 단계에서 진행한다.
- 이번 계획의 목표는 실제 백엔드 없이도 사용자 흐름을 검증할 수 있는 프론트엔드 완성도 보강이다.

## 제외 범위

- 데이터 내보내기/가져오기
- 실제 OAuth provider 연동
- 운영 DB 저장소 전환
- 성경전서 전체 본문 seed
- 고품질 서버 TTS
- 오프라인/PWA 캐싱

## 우선순위 요약

| 우선순위 | 기능 | 이유 |
| --- | --- | --- |
| P0 | 통독 플랜, 현재 절 자동 추적 | 사용자가 다음에 무엇을 읽을지 바로 알 수 있게 한다. |
| P0 | 다중 구절 선택, 모바일 액션 UX | 리더의 핵심 작업 속도를 높인다. |
| P1 | TTS 재생목록 개선 | 듣기 중심 사용자의 반복 사용성을 높인다. |
| P1 | 필터/정렬 고도화, 개인 노트 | 저장 데이터가 늘어났을 때 다시 찾고 활용하게 한다. |
| P1 | 대시보드 개선, 키보드 단축키 | 앱 첫 화면과 고빈도 조작을 정리한다. |

## Phase FE-01: 통독 플랜과 현재 절 자동 추적

### 목표

대시보드와 리더가 "오늘 읽을 분량"과 "지금 읽고 있는 절"을 명확히 안내하게 한다.

### 포함 기능

- 통독 플랜 템플릿: 1년, 6개월, 90일, 신약 30일
- 오늘 읽을 범위 계산
- 플랜 시작일, 목표일, 일일 분량 저장
- 현재 보이는 절 자동 감지
- 마지막 읽은 절과 스크롤 위치 저장 정밀도 개선
- 이어 읽기 버튼을 현재 절 기준으로 개선

### 구현 체크리스트

- [x] `ReadingPlan` 타입을 추가한다.
- [x] `UserDataState`에 활성 통독 플랜 상태를 추가한다.
- [x] 기존 localStorage 저장소 마이그레이션을 추가한다.
- [x] 플랜 템플릿별 일일 장 범위 계산 함수를 만든다.
- [x] 대시보드에 오늘 읽을 범위 카드를 추가한다.
- [x] 리더에서 `IntersectionObserver`로 현재 절을 추적한다.
- [x] 현재 절 변경 시 progress를 debounce 저장한다.
- [x] 이어 읽기 동작을 마지막 절 중심으로 조정한다.
- [x] 플랜 완료/건너뜀/재계산 UI를 최소 기능으로 제공한다.

### 수용 기준

- [x] 사용자는 대시보드에서 오늘 읽을 범위를 확인할 수 있다.
- [x] 리더 스크롤 후 새로고침하면 마지막으로 보던 절 근처로 복원된다.
- [x] 통독 플랜이 없어도 기존 읽기 흐름은 깨지지 않는다.

## Phase FE-02: 다중 구절 선택과 모바일 액션 UX

한 절씩만 조작하는 흐름을 보완해 여러 구절을 한 번에 복사, 강조, 인용 저장, TTS 재생할 수 있게 한다.

### 포함 기능

- 다중 구절 선택 모드
- 연속 범위 선택
- 선택 구절 일괄 복사
- 선택 구절 일괄 강조
- 선택 구절 인용 저장
- 모바일 하단 액션 시트

### 구현 체크리스트

- [x] `selectedVerseIds` 상태를 추가한다.
- [x] 단일 선택과 다중 선택 모드를 분리한다.
- [x] 절 항목에 선택 상태와 체크 UI를 추가한다.
- [x] 첫 절과 마지막 절을 기준으로 범위 선택을 구현한다.
- [x] 선택 구절 복사 형식을 정의한다.
- [x] 선택 구절 일괄 강조 색상 선택 UI를 만든다.
- [x] 선택 구절 인용 저장 모달을 기존 목록 모달과 재사용한다.
- [x] 모바일 폭에서는 절 액션을 하단 시트로 표시한다.
- [x] 선택 취소, 전체 선택 해제, 선택 개수 표시를 제공한다.

### 수용 기준

- [x] 사용자는 여러 절을 선택해 한 번에 복사할 수 있다.
- [x] 사용자는 선택한 여러 절을 같은 목록에 인용 저장할 수 있다.
- [x] 모바일 화면에서 액션 UI가 본문을 과도하게 가리지 않는다.

## Phase FE-03: TTS 재생목록 개선

### 목표

TTS를 단순 재생 버튼이 아니라 짧은 성경 오디오 플레이어처럼 사용할 수 있게 한다.

### 포함 기능

- 재생 큐: 현재 장, 선택 범위, 오늘 읽을 범위
- 이전/다음 절 안정화
- 반복 재생 옵션
- 자동 스크롤 옵션
- 재생 중 구절 위치 표시 개선
- TTS 음성/속도 프리셋 개선

### 구현 체크리스트

- [x] TTS 상태를 `idle`, `playing`, `paused`, `error`로 정리한다.
- [x] 재생 큐 타입을 추가한다.
- [x] 현재 장 전체 재생을 큐 기반으로 바꾼다.
- [x] 선택 구절/범위 재생을 큐에 연결한다.
- [x] 오늘 읽을 범위 재생 버튼을 추가한다.
- [x] 반복 재생 토글을 추가한다.
- [x] 재생 중 현재 절로 자동 스크롤하는 옵션을 추가한다.
- [x] TTS 미지원 브라우저 안내 상태를 정리한다.

### 수용 기준

- [x] 사용자는 현재 장 또는 선택 범위를 끊김 없이 들을 수 있다.
- [x] TTS가 읽는 절은 화면에서 명확히 표시된다.
- [x] 절 번호/참조는 음성으로 읽지 않고 본문만 읽는다.

## Phase FE-04: 필터/정렬 고도화와 개인 노트

### 목표

강조, 인용, 노트가 늘어나도 사용자가 필요한 구절을 빠르게 찾을 수 있게 한다.

### 포함 기능

- 강조 목록 색상/성경 권/태그 필터
- 인용 목록 내 검색
- 인용 목록 정렬: 최근 저장순, 성경 순서, 자주 사용순
- 장별 노트
- 구절별 노트
- 노트가 있는 장/절 표시

### 구현 체크리스트

- [x] `StudyNote` 타입을 추가한다.
- [x] `UserDataState`에 노트 배열을 추가한다.
- [x] localStorage 마이그레이션을 추가한다.
- [x] 리더에 장 노트 버튼을 추가한다.
- [x] 절 액션에 구절 노트 편집을 추가한다.
- [x] 강조 페이지에 색상/권 필터를 추가한다.
- [x] 인용 페이지에 목록 내 검색과 정렬 UI를 추가한다.
- [x] 노트가 있는 절에 시각 표시를 추가한다.
- [x] 빈 상태와 검색 결과 없음 상태를 구분한다.

### 수용 기준

- [x] 사용자는 장/절 단위 노트를 저장하고 다시 열 수 있다.
- [x] 인용 구절이 많아도 검색과 정렬로 필요한 항목을 찾을 수 있다.
- [x] 강조 목록은 색상과 성경 권 기준으로 필터링된다.

## Phase FE-05: 대시보드와 빠른 조작 개선

### 목표

앱 첫 화면에서 다음 행동이 분명하게 보이고, 반복 조작은 빠르게 수행할 수 있게 한다.

### 포함 기능

- 오늘 읽을 분량 카드
- 최근 읽은 위치 타임라인
- 최근 강조/인용/노트 통합 활동
- 권별 진척도 요약 개선
- 키보드 단축키
- 간단 명령 팔레트

### 구현 체크리스트

- [x] 대시보드 정보를 `continue`, `today`, `activity`, `progress` 영역으로 재정리한다.
- [x] 최근 활동 모델을 읽기/강조/인용/노트 이벤트로 통합 표시한다.
- [x] 권별 진척도에서 미완료 장으로 이동하는 액션을 추가한다.
- [x] `/`, `j`, `k`, `n`, `p`, `space` 등 기본 단축키를 정의한다.
- [x] 단축키 도움말 모달을 만든다.
- [x] 검색/이동/뷰 전환용 명령 팔레트를 추가한다.
- [x] 키보드 조작이 입력창 포커스와 충돌하지 않게 처리한다.

### 수용 기준

- [x] 사용자는 대시보드에서 오늘 읽을 분량과 이어 읽기 위치를 즉시 확인한다.
- [x] 사용자는 최근 활동에서 저장한 구절이나 노트로 이동할 수 있다.
- [x] 기본 키보드 단축키가 리더 사용을 방해하지 않고 동작한다.

## 실행 순서

1. Phase FE-01을 먼저 구현한다. 통독 플랜과 현재 절 추적은 대시보드, TTS, 읽기 기록의 기준 데이터가 된다.
2. Phase FE-02를 구현한다. 다중 선택과 모바일 액션 UX는 리더의 핵심 조작 구조를 안정화한다.
3. Phase FE-03을 구현한다. TTS 큐가 통독 플랜과 다중 선택 범위를 재사용하게 한다.
4. Phase FE-04를 구현한다. 저장 데이터가 늘어나는 시점에 필터/정렬/노트를 붙인다.
5. Phase FE-05를 구현한다. 앞선 기능들을 대시보드와 빠른 조작으로 연결한다.
